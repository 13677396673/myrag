## 一、文档目的

本文档是 RAG 智能知识库系统的概要设计说明书，用于：
1. 划分系统模块，明确各模块的职责边界
2. 定义模块间的接口关系与数据流
3. 设计数据库表结构
4. 规划前后端 API 接口
5. 为核心业务流程提供时序说明

本文档的读者为系统开发人员。

### 2.2 模块详细定义

#### 2.2.1 表示层 — Web 前端

| 模块 | 职责 | 主要页面/组件 | 依赖 |
|------|------|--------------|------|
| **用户模块** | 登录、注册、注销、个人信息管理 | 登录页、注册页、个人信息页 | API 接口层 |
| **文档管理模块** | 文档上传、列表展示、状态跟踪、删除、详情查看 | 文档列表页、文档上传组件、文档详情页 | API 接口层 |
| **对话模块** | 聊天界面、流式消息展示、引用来源展示、历史对话切换 | 对话页、消息组件、来源引用组件 | API 接口层 |
| **数据集模块** | 知识库的创建、管理、列表 | 数据集列表页、数据集详情页 | API 接口层 |
| **管理员面板** | 用户管理、系统信息查看 | 管理员面板页 | API 接口层 |

#### 2.2.2 接口层 — FastAPI

| 模块 | 职责 |
|------|------|
| **路由注册** | 注册所有业务路由，统一前缀 `/api/v1` |
| **JWT 鉴权中间件** | 验证请求 Token，解析当前用户 |
| **CORS 中间件** | 允许前端跨域请求 |
| **请求日志中间件** | 记录请求耗时、状态码 |
| **全局异常处理** | 统一错误响应格式 |

#### 2.2.3 业务逻辑层

| 模块 | 职责 | 核心类/接口 |
|------|------|-------------|
| **用户服务 (UserService)** | 用户注册、登录验证、密码加密、JWT 生成与校验、用户信息 CRUD | `UserService`, `AuthService` |
| **文档服务 (DocumentService)** | 文档上传存储、文档 CRUD、处理状态管理、触发异步解析任务 | `DocumentService` |
| **数据集服务 (DatasetService)** | 知识库 CRUD、文档与知识库的关联管理 | `DatasetService` |
| **对话服务 (ConversationService)** | 对话 CRUD、消息管理、上下文组装 | `ConversationService` |
| **管理后台服务 (AdminService)** | 用户管理、系统统计信息 | `AdminService` |
| **RAG 引擎** | 见下节 | 见下节 |

#### 2.2.4 RAG 引擎（核心领域层）

RAG 引擎是系统的核心，严格遵循**接口抽象**原则，所有组件通过依赖注入组合。

| 引擎组件 | 职责 | 抽象接口 | 内置实现 |
|----------|------|----------|----------|
| **文档解析器 (Parser)** | 将各类格式文件解析为纯文本+元数据 | `DocumentParser` | `PDFParser`, `TextParser`, `DocxParser`, `PPTXParser`, `XLSXParser`, `OCRParser`, `WebParser` |
| **文本切片器 (Splitter)** | 将长文本按策略切分为片段 | `TextSplitter` | `FixedSizeSplitter`, `MarkdownSplitter`, `SemanticSplitter` |
| **Embedding 引擎** | 将文本转为向量 | `EmbeddingBackend` | `BGESmallEmbedding`, `OpenAIEmbedding`, `DeepSeekEmbedding` |
| **向量数据库** | 存储向量并支持相似度检索 | `VectorStore` | `ChromaDBStore`, `FAISSStore`, `MilvusStore`, `PGVectorStore` |
| **检索器 (Retriever)** | 根据 Query 检索相关文档片段、支持重排序 | `Retriever` | `VectorRetriever`, `BM25Retriever`, `HybridRetriever` |
| **生成器 (Generator)** | 调用 LLM 生成最终回答 | `LLMBackend` | `DeepSeekLLM`, `OpenAILLM`, `OllamaLLM` |

#### 2.2.5 基础设施层

| 模块 | 职责 | 抽象接口 | 内置实现 |
|------|------|----------|----------|
| **任务队列 (TaskQueue)** | 异步执行耗时任务（文档解析、Embedding 等） | `TaskQueueBackend` | `HueyQueue`（默认）、`CeleryQueue`、`ARQQueue` |
| **文件存储 (FileStorage)** | 统一管理上传文件的存储与读取 | `FileStorageBackend` | `LocalFileStorage`（默认）、`S3FileStorage`、`MinioFileStorage` |
| **配置管理 (Config)** | 分层加载配置，支持多环境 | — | `.env` + `config.yaml`，环境变量优先级最高 |
| **关系数据库** | 存储用户、文档、对话等结构化数据 | — | SQLAlchemy ORM，SQLite（开发）/ PostgreSQL（生产） |

### 2.3 模块依赖关系

```
用户模块 (UserService)
  ├── 依赖: 关系数据库 (SQLAlchemy)
  └── 依赖: 配置管理

文档管理模块 (DocumentService)
  ├── 依赖: 关系数据库
  ├── 依赖: 文件存储 (FileStorage)
  ├── 依赖: 任务队列 (TaskQueue) — 触发异步解析
  └── 依赖: 配置管理

数据集模块 (DatasetService)
  ├── 依赖: 关系数据库
  ├── 依赖: 文档管理模块 (DocumentService) — 获取文档
  └── 依赖: 配置管理

对话模块 (ConversationService)
  ├── 依赖: 关系数据库
  ├── 依赖: RAG 引擎（检索器 + 生成器）
  └── 依赖: 配置管理

RAG 引擎
  ├── 文档解析器 → 依赖: 文件存储（读取文件）
  ├── 文档解析器 → 依赖: 任务队列（异步执行）
  ├── 文本切片器 → 依赖: 文档解析器（输出）
  ├── Embedding 引擎 → 依赖: 配置管理（选择模型）
  ├── 向量数据库 → 独立模块，通过接口调用
  ├── 检索器 → 依赖: Embedding 引擎 + 向量数据库
  └── 生成器 → 依赖: LLMBackend + 配置管理

管理后台模块 (AdminService)
  ├── 依赖: 用户模块
  └── 依赖: 关系数据库
```

**模块间的禁止反向依赖**：
- RAG 引擎**不依赖**任何业务 Service（如 DocumentService）
- 基础设施层**不依赖**业务逻辑层
- 前端模块**只依赖** API 接口

## 三、数据库设计

### 3.1 ER 图（文本描述）

```
User 1 ──── * Dataset          # 用户拥有多个数据集
User 1 ──── * Document         # 用户上传多个文档
User 1 ──── * Conversation     # 用户有多个对话
Dataset 1 ──── * Document      # 一个数据集包含多个文档（多对多关系通过关联表）
Document * ──── 1 Dataset     # 一个文档属于一个数据集
Document 1 ──── * Chunk        # 一个文档被切分为多个 Chunk
Conversation 1 ──── * Message  # 一个对话有多条消息
Chunk * ──── * Message         # 一条消息引用多个 Chunk（来源引用）
```

### 3.2 表结构定义

#### user（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK, 自动生成 | 用户唯一标识 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(255) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 密码哈希 |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'user' | 角色：user / admin |
| is_active | BOOLEAN | DEFAULT TRUE | 账号是否激活 |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

#### dataset（数据集/知识库表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 数据集唯一标识 |
| name | VARCHAR(255) | NOT NULL | 数据集名称 |
| description | TEXT | 可选 | 数据集描述 |
| user_id | UUID | FK → user.id, NOT NULL | 所属用户 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

#### document（文档表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 文档唯一标识 |
| filename | VARCHAR(255) | NOT NULL | 原始文件名 |
| file_type | VARCHAR(20) | NOT NULL | 文件类型：pdf/docx/pptx/xlsx/txt/md/png/jpg/url |
| file_size | BIGINT | NOT NULL | 文件大小（字节） |
| file_path | VARCHAR(500) | NOT NULL | 存储路径（对应 FileStorage 路径） |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'pending' | pending → parsing → indexing → completed / failed |
| error_message | TEXT | 可选 | 处理失败时的错误信息 |
| dataset_id | UUID | FK → dataset.id, 可选 | 所属数据集 |
| user_id | UUID | FK → user.id, NOT NULL | 上传者 |
| chunk_count | INT | DEFAULT 0 | 切片数量 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

**status 枚举值**：`pending` → `parsing` → `splitting` → `indexing` → `completed` | `failed`

#### chunk（文档切片表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 切片唯一标识 |
| document_id | UUID | FK → document.id, NOT NULL | 所属文档 |
| content | TEXT | NOT NULL | 切片文本内容 |
| chunk_index | INT | NOT NULL | 在文档中的序号 |
| metadata | JSONB | 可选 | 额外元数据（页码、标题等） |
| vector_id | VARCHAR(255) | 可选 | 向量数据库中的 ID（关联用） |
| created_at | TIMESTAMP | NOT NULL | |

#### conversation（对话表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 对话唯一标识 |
| title | VARCHAR(255) | NOT NULL | 对话标题（自动生成或用户命名） |
| user_id | UUID | FK → user.id, NOT NULL | 所属用户 |
| dataset_id | UUID | FK → dataset.id, 可选 | 关联的数据集（为空则检索全部文档） |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

#### message（消息表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 消息唯一标识 |
| conversation_id | UUID | FK → conversation.id, NOT NULL | 所属对话 |
| role | VARCHAR(20) | NOT NULL | 角色：user / assistant / system |
| content | TEXT | NOT NULL | 消息内容 |
| metadata | JSONB | 可选 | LLM 返回的额外信息（token 用量等） |
| created_at | TIMESTAMP | NOT NULL | |

#### message_chunk（消息-切片关联表，来源引用）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | |
| message_id | UUID | FK → message.id, NOT NULL | assistant 消息 |
| chunk_id | UUID | FK → chunk.id, NOT NULL | 引用的文档切片 |
| relevance_score | FLOAT | NOT NULL | 相关性得分 |
| created_at | TIMESTAMP | NOT NULL | |

### 3.3 向量数据库数据模型

向量数据库（ChromaDB 为例）中每条记录的结构：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | 与 chunk.id 一致 |
| vector | List[float] | Embedding 向量（BGE-small-zh 为 384 维） |
| metadata | dict | 包含：chunk_id, document_id, dataset_id, user_id, chunk_index, content_preview（前 200 字符用于检索结果显示）, page_number 等 |

## 四、API 接口概览

### 4.1 接口规范

- **基础路径**：`/api/v1`
- **响应格式**：
  ```json
  // 成功
  { "code": 0, "data": {...}, "message": "ok" }
  // 失败
  { "code": -1, "data": null, "message": "错误描述" }
  ```
- **认证方式**：`Authorization: Bearer <JWT Token>`
- **分页请求**：`?page=1&page_size=20`
- **分页响应**：`{ "code": 0, "data": { "items": [...], "total": 100, "page": 1, "page_size": 20 } }`

### 4.2 接口列表

#### 4.2.1 认证接口

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | `/api/v1/auth/register` | 用户注册 | ❌ |
| POST | `/api/v1/auth/login` | 用户登录，返回 JWT | ❌ |
| POST | `/api/v1/auth/refresh` | 刷新 Token | ✅ |
| POST | `/api/v1/auth/logout` | 注销 | ✅ |

#### 4.2.2 用户接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/users/me` | 获取当前用户信息 |
| PUT | `/api/v1/users/me` | 更新个人信息 |
| PUT | `/api/v1/users/me/password` | 修改密码 |

#### 4.2.3 数据集接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/datasets` | 获取数据集列表 |
| POST | `/api/v1/datasets` | 创建数据集 |
| GET | `/api/v1/datasets/{id}` | 获取数据集详情 |
| PUT | `/api/v1/datasets/{id}` | 更新数据集信息 |
| DELETE | `/api/v1/datasets/{id}` | 删除数据集（及内部文档） |

#### 4.2.4 文档接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/datasets/{dataset_id}/documents` | 获取数据集内文档列表 |
| POST | `/api/v1/datasets/{dataset_id}/documents` | 上传文档（multipart） |

| GET | `/api/v1/documents/{id}/status` | 获取文档处理状态 |

#### 4.2.5 对话接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/conversations` | 获取对话列表 |
| POST | `/api/v1/conversations` | 创建新对话 |
| GET | `/api/v1/conversations/{id}` | 获取对话详情 |
| DELETE | `/api/v1/conversations/{id}` | 删除对话 |
| GET | `/api/v1/conversations/{id}/messages` | 获取对话消息列表 |

#### 4.2.6 管理后台接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/v1/admin/users` | 用户列表 | admin |
| GET | `/api/v1/admin/stats` | 系统统计信息 | admin |

### 4.3 SSE 消息流格式

```
data: {"type": "delta", "content": "正在"}    # LLM 生成的文本片段
data: {"type": "delta", "content": "回答中"}
data: {"type": "done", "content": ""}          # 生成完成
data: {"type": "sources", "content": [         # 引用来源
  {"chunk_id": "xxx", "content": "...", "document_name": "xxx.pdf", "score": 0.95}
]}
data: {"type": "error", "content": "错误描述"}  # 错误信息
```

### 6.1 文档上传与处理流程

```

 |                       |                      |                       |                  |                    |
 |  1. 上传文件           |                      |                       |                  |                    |
 |──────────────────────>|                      |                       |                  |                    |
 |                       |  2. POST /documents   |                       |                  |                    |
 |                       |─────────────────────>|                       |                  |                    |
```

### 7.3 检索策略

**默认检索策略**：向量检索 + Top-K 重排序

1. 将用户问题通过 EmbeddingBackend 转为向量
2. 在 VectorStore 中检索 Top-20（余弦相似度）
3. 根据用户 ID 过滤（只能看到自己的文档）
4. 可选：根据 dataset_id 过滤
5. 返回 Top-5 作为上下文

**可扩展检索模式**（Phase 3 实现）：
- **混合检索**：向量检索（70%）+ BM25 关键词检索（30%），结果融合
- **MMR**：最大边际相关性，提高结果多样性
- **Reranker**：使用交叉编码器重排序（如 BAAI/bge-reranker-v2-m3）

## 十一、RAG 引擎接口定义（详细）

### 11.1 DocumentParser 接口

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import List, Optional

@dataclass
class ParsedDocument:
    """解析器输出：原始文本 + 结构化元数据"""
    content: str                              # 完整文本内容
    metadata: dict = field(default_factory=dict)  # {title, author, page_count, ...}
    sections: List[dict] = field(default_factory=list)  # [{"heading": "xxx", "content": "xxx", "page": 1}, ...]

class DocumentParser(ABC):
    """文档解析器抽象接口"""
    
    @abstractmethod
    def parse(self, file_path: str) -> ParsedDocument:
        """解析文件为结构化文本"""
        pass
    
    @classmethod
    @abstractmethod
    def supported_extensions(cls) -> List[str]:
        """返回支持的文件扩展名列表，如 ['.pdf', '.PDF']"""
        pass
```

### 11.2 TextSplitter 接口

```python
@dataclass
class DocumentChunk:
    """切片输出"""
    content: str
    chunk_index: int
    metadata: dict = field(default_factory=dict)  # {document_id, page_number, heading, ...}

class TextSplitter(ABC):
    """文本切片器抽象接口"""
    
    @abstractmethod
    def split(self, text: str, metadata: dict) -> List[DocumentChunk]:
        """将文本按策略切分为片段列表"""
        pass
    
    @property
    @abstractmethod
    def type_name(self) -> str:
        """切片器类型名称，用于配置识别"""
        pass
```

### 11.3 EmbeddingBackend 接口

```python
class EmbeddingBackend(ABC):
    """Embedding 模型抽象接口"""
    
    @abstractmethod
    def embed_text(self, text: str) -> List[float]:
        """将单条文本转为向量"""
        pass
    
    @abstractmethod
    def embed_documents(self, texts: List[str]) -> List[List[float]]:
        """批量将文本转为向量（可优化批处理性能）"""
```
